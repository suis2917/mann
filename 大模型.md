**RAG** 是一种结合 **信息检索（Retrieval）** 和 **文本生成（Generation）** 的技术，通过以下两步生成更准确、可靠的回答：

1. **检索**：从外部知识库（如文档、数据库）中查找与问题相关的信息片段。
2. **生成**：基于检索到的内容，由大语言模型（如GPT）生成最终回答。

#### **知识库（Knowledge Base） vs 知识图谱（Knowledge Graph）核心区别**

|   **维度**   |                   **知识库**                    |                       **知识图谱**                        |
| :----------: | :---------------------------------------------: | :-------------------------------------------------------: |
|   **结构**   |     以文档、表格或问答对形式存储，松散关联      |         用节点（实体）和边（关系）组成的语义网络          |
| **数据组织** |              基于关键词或分类检索               |       基于实体间的逻辑关系推理（如“A是B的母公司”）        |
| **技术实现** |   数据库/文档管理系统（如MySQL、Confluence）    |             图数据库（如Neo4j）、RDF/OWL标准              |
| **查询方式** |              关键词搜索、模糊匹配               |    语义查询、路径分析（如“找出所有间接关联的供应商”）     |
| **适用场景** | • 存储静态知识（如产品手册） • 快速检索明确答案 | • 复杂关系推理（如金融风控） • 智能推荐（如电商商品关联） |
|  **AI结合**  |                 作为RAG的检索源                 |           支撑语义理解、推理型AI（如智能问答）            |
|   **例子**   |             • 企业Wiki • 客服FAQ库              |      • Google搜索引擎的“知识卡片” • 医疗疾病关系网络      |

### **Embedding向量库（Embedding Vector Database）简明定义**

**Embedding向量库** 是一种专门用于存储、索引和快速检索**高维向量数据**的数据库，核心功能是为AI模型（如大语言模型、推荐系统）提供高效的**语义相似度搜索**能力。

------

### **核心特性**

1. **向量化存储**
   - 将文本、图像、音频等数据转换为**固定维度的数值向量**（如768维、1024维）。
2. **相似度检索**
   - 支持近邻搜索（ANN），快速找到与查询向量最相似的条目。
3. **AI原生支持**
   - 直接对接Embedding模型（如BERT、CLIP），实现“输入文本→向量→检索”的端到端流程。
